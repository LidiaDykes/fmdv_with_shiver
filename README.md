# fmdv_with_shiver
(not only) FMDV de novo assembly from paired-end Illumina reads

```fmdv_with_shiver``` pipeline assemblies Whole Genome Sequence (WGS) of foot-and-mouth disease virus (FMDV) from Illumina paired-end reads. The pipeline can be adapted to assemble genome of any other virus if Multiple Sequence Alignment (MSA) of reference genomes is provided.


The ```fmdv_with_shiver``` is based on ```shiver``` developed by Chris Wymant (@ChrisHIV, https://github.com/ChrisHIV/shiver). In addition of WGS, shiver produces variants report in a csv format (```sample_name_remap_BaseFreqs.csv```). ```fmdv_with_shiver``` allows for automated processing of all samples in a single directory, from data organisation to coverage plots. In most cases, it performs well at FMDV assembly; however, for samples with poor coverage, it will fill the gaps with the closest reference from the MSA, which might not be correct. Such cases can be spotted from the multi-panel coverage plot generated by ``` plot_multipanel_coverage.py```. If coverage is near zero for genomic regions outside of the poly(C) tract, it is recommended to ensure that reconstructed genome is correct by viewing mapped reads in the Integrative Genomics Viewer (IGV, https://igv.org/) or equivalent. 


Originally written to analyse data of my PhD student. Published here in case useful for others working on FMDV or other RNA viruses. If you happen to use it for published work, please cite the link to this repo.

## 1. Installation

### 1.a. Install miniconda
The quickest way to install all the dependencies for this pipeline is via conda. For most users, a minimal version of anaconda (miniconda) is sufficient (more info: https://www.anaconda.com/docs/getting-started/miniconda/main ). 


To install the latest miniconda on Linux computer follow the installation steps described here: https://www.anaconda.com/docs/getting-started/miniconda/install/linux-install#installation-steps.


Don’t forget to verify your installation: https://www.anaconda.com/docs/getting-started/miniconda/install/linux-install#verify-your-install. 

### 1.b. Clone this repository
```
git clone https://github.com/LidiaDykes/fmdv_with_shiver.git
```

### 1.c. Install dependencies via conda
You can install all the dependencies required for this pipeline via conda, using provided ```environment_fmdv_with_shiver.yml``` file. 

```
# enter the downloaded fmdv_with_shiver directory 
cd fmdv_with_shiver

# create conda environment named fmdv_with_shiver
conda env create -f environment_fmdv_with_shiver.yml

# OPTION: if you wish to save the conda environment under a different name
conda env create -f environment_fmdv_with_shiver.yml -n my_new_env

# check if the new environment was installed
conda env list
```


### 1.d. Make scripts executable:
```
# enter the src directory
cd src

# make scripts executable

chmod u+x fix_fqs_templ.py
chmod u+x fetch_Fastqscreen_genomes.bsh
chmod u+x organise_fastq_files.bsh
chmod u+x de_novo_with_shiver.bsh
chmod u+x plot_multipanel_coverage.py
```


### 1.e. Specify path to the scripts
To call the scripts you can either:
-	each time when running scripts, specify the absolute path to the ```fmdv_with_shiver/src/script```. For those not familiar with the Linux command line, this might be the simplest option. Use ```pwd``` command while located in ``` fmdv_with_shiver/src ``` directory to identify the absolute path.
-	add the fmdv_with_shiver/src to your ```$PATH``` and reload the linux session  

```
# add the fmdv_with_shiver/src to your $PATH
# where ‘\’ is used to divide single line of code into multiple lines

if ! grep -q '/absolute_path_to/fmdv_with_shiver/src' ~/.bashrc; then \
    echo 'export PATH="$PATH:/absolute_path_to/fmdv_with_shiver/src"' >> ~/.bashrc; \
fi; \
source ~/.bashrc
```

### 1.f. Activate conda environment
```
conda activate fmdv_with_shiver
```

### 1.g. Uncompress common contaminants data
``` fmdv_with_shiver/data``` directory contains data files required to run ```fmdv_with_shiver```; these are:
* A MSA of FMDV isolates which should cover genetic diversity of FMDV. This MSA can be replaced by a fasta file representing genetic diversity of a virus of choice; 
* ```MyAdapters.fasta``` contains adapters used by Logan *et al.* 2014 (doi: 10.1186/1471-2164-15-828). This file with adapters should be replaced accordingly;
* ```MyPrimers.fasta``` contains primers used by Logan *et al.* 2014 (doi: 10.1186/1471-2164-15-828). This file with primers should be replaced accordingly;
* ```FastQ_Screen_Genomes.tar.gz``` genomic data of common contaminants obtained from https://www.bioinformatics.babraham.ac.uk/ as part of ```FastQ Screen --get_genomes``` option, which seems to no longer work due to data relocation.
* ```contaminant_genomes.csv``` a list of host genomes used to build host genomes database in ``` FastQ_Screen_Genomes``` and used by ```FastQ Screen``` (see section below). 

To be able to filter out the common contaminants out of your fastq files, you need to first uncompress the ```FastQ_Screen_Genomes.tar.gz ```.
If you would like to store genomic sequences of common contaminants elsewhere (e.g. due limits in storage), move the compressed file to your chosen location first.

```
# enter the fmdv_with_shiver/data
cd ../data

# OPTIONAL: move the compressed file to a new location
mv FastQ_Screen_Genomes.tar.gz /path/to/the/new/location

# uncompress data directory
tar -xzvf FastQ_Screen_Genomes.tar.gz
```

### 1.h. Download host genomes
```FastQ Screen``` (implemented in ```fmdv_with_shiver```) can also remove host reads which can be common in gunshot sequencing. ```fmdv_with_shiver/data``` contains ```contaminant_genomes.csv``` which lists names and RefSeq numbers of common FMDV livestock hosts. ```contaminant_genomes.csv``` is used by ```fetch_Fastqscreen_genomes.bsh``` to generate indexed host genome data inside ```FastQ_Screen_Genomes```, which are subsequently used by ```FastQ Screen```. 

If you moved the ```FastQ_Screen_Genomes.tar.gz``` into a new location, you need to specify the path to the new location with ```--fqs_genomes ``` when running ```fetch_Fastqscreen_genomes.bsh```; the script was written to save host genomes inside ```FastQ_Screen_Genomes```.

```contaminant_genomes.csv``` can be edited to include hosts of choice (species names and RefSeq numbers obtained from https://www.ncbi.nlm.nih.gov/datasets/genome/). If you modified the ``` contaminant_genomes.csv``` and saved it under a different name, run ``` fix_fqs_templ.py``` to update ```fastq_screen.conf.template``` file to ```fastq_screen.conf.template.new```, and specify the modified csv when running ```fetch_Fastqscreen_genomes.bsh``` using ```--host_info``` flag.

```
# OPTIONAL: if you customised contaminant_genomes.csv
fix_fqs_templ.py updated_contaminant_genomes.csv


# build host genome data in FastQ_Screen_Genomes

fetch_Fastqscreen_genomes.bsh \
[--fqs_genomes /absolute/path/to/the/FastQ_Screen_Genomes] \
[--host_info updated_contaminant_genomes.csv]
```


## 2. Running fmdv_with_shiver 

### 2.a. Organise your paired-end reads into sample-specific subdirectories 
Given the path to a directory containing your paired-end reads data, ```organise_fastq_files.bsh``` will create a symlink to your data and create in your current directory a subdirectory called ```analysis_date```. For the script to work, all paired-end fastq files (R1 and R2) need to be located in a single directory (with no additional subdirectories).

```
organise_fastq_files.bsh /path/to/your_Illumina_data name_of_symlink
```

### 2b. De novo assemble reads into WGS
This step will run ```de_novo_with_shiver.bsh``` which will:
* quality filter (QC) all reads:
    * trim 20 nucleotides from the 5’ end;
    * trim 5 nucleotides from the 3’ end;
    * keep reads with min. Phred quality score 30;
    * keep reads with min. read length 70 nucleotides;
    * trim regions containing Ns;
* remove duplicates;
* assemble contigs based on quality-filtered reads (required for ```shiver```);
* run ```shiver``` on raw reads (shiver has its own QC filtering, see https://github.com/ChrisHIV/shiver) to generate de novo consensus;
* filter out common contaminants, such as host reads;
* using ```bwa-mem```, map QC-filtered and decontaminated reads to the ```shiver```-assembled genome, to verify correctness of the ```shiver``` assembly. 


Enter the ```analysis_date``` directory:
```
cd analysis_date
```


You should see following directory architecture:
```
analysis_date
 |-- Sample_01
 |  |-- Sample_01_R1_001.fastq.gz
 |  |-- Sample_01_R2_001.fastq.gz
 |-- Sample_02
 |-- Sample_03
```


```fmdv_with_shiver``` was written for fastq files with read R1 and R2 marked as ```1:N:0``` (R1) and ```2:N:0``` (R2). ```shiver``` (https://github.com/ChrisHIV/shiver) accepts fastq files with R1 and R2 identified as follows: ```/1``` (R1) and ```/2``` (R2). ```de_novo_with_shiver.bsh``` will automatically change identifiers to shiver-readable format as long as the original read identifier follows format ```@read_ID<while space>read_number```. If your format is different, then please modify lines 125-126 of ``` de_novo_with_shiver.bsh ``` accordingly. 


Again, if you moved the ```FastQ_Screen_Genomes.tar.gz``` to a new location, you need to specify that location as your last argument when running ```de_novo_with_shiver.bsh```. 

```
# Run assembly and specify non-default parameters using appropriate flags
de_novo_with_shiver.bsh \
[--fasta virus_reference_database.fasta] \
[--adapters Custom_Adapters.fasta] \
[--primers Custom_Primers.fasta] \
[--fqs_genomes /path/to/the/FastQ_Screen_Genomes] \
[--fqs_template fmdv_with_shiver/src/fastq_screen.conf.template.new]
```


You can assemble FMDV reads with default FMDV data, adapters, primers and FastQ_Screen genomic data in the original location by running de_novo_assembly.bsh without any arguments.

```
# running de_novo_shiver.bsh for FMDV with default parameters
de_novo_with_shiver.bsh
``` 


When finished, each sample directory should contain:
* consensus called: ```sample_name_shiver_final_ref.fasta```;
* variant list called: ```sample_name _remap_BaseFreqs.csv```;
* binary alignment files called: ```sample_name_pe.sorted.bam```, ```sample_name_pe.sorted.bam.bai``` (for viewing in IGV).


### 2.c. Plot coverage for all samples
Once ```de_novo_with_shiver.bsh``` finished, run the plotting script from the ```analysis_date``` directory.

```
plot_multipanel_coverage.py
```
This will return a ```date_bwa_remapped_coverage_multipanel.png``` showing coverage for all samples.
